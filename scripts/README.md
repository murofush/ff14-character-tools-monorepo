# scripts/README

スクリプトは用途別に分類して管理する。新規追加は該当ディレクトリに配置する。

## ディレクトリ構成

- `ai/`: AI クライアント起動・レビュー支援
- `dev/`: ローカル開発補助
- `qa/`: 疎通チェック・E2E
- `ops/`: 運用系（監視/アラート/メンテ）

## 代表的な呼び出し（例）

- 簡易疎通: `make smoke`（`scripts/qa/smoke.sh`）
- 開発起動: `make dev` または `pnpm dev`
- CI 相当検証: `make verify` または `pnpm run verify`

## 注意点

- スクリプトは環境変数で上書き可能にし、ハードコードを避ける
- 失敗時のエラーメッセージは次の行動が分かる内容にする
- 本番環境を操作する処理は、明示的な実行手順と安全策を必須化する
